# Чек-лист переменных окружения и ключей

**Главное: приложение полностью работает на mock-данных без единой переменной
ниже. Ничто в этом списке не блокирует текущую работу с прототипом** — добавляйте
ключи постепенно, по одному, в любом порядке и темпе, когда будете готовы
подключать конкретную реальную интеграцию. Файл-шаблон — `.env.example` в корне
репозитория; рабочую копию для реальных значений нужно положить в `.env.local`
(не коммитить в git).

## База данных (Supabase / Postgres)

| Переменная | Что разблокирует | Где получить |
|---|---|---|
| `SUPABASE_URL` | URL проекта Supabase — без этого вся БД остаётся mock-данными в `lib/mock.ts` | Supabase Dashboard → Project Settings → API |
| `SUPABASE_ANON_KEY` | Публичный клиентский ключ Supabase | Supabase Dashboard → Project Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | Серверный ключ с полным доступом (обходит RLS) — хранить только на сервере, никогда не светить в клиенте | Supabase Dashboard → Project Settings → API |
| `DATABASE_URL` | Прямая строка подключения к Postgres (для миграций/инструментов вне Supabase SDK) | Supabase Dashboard → Project Settings → Database |

**Стоимость:** у Supabase есть бесплатный тариф (Free tier), достаточный для
старта и тестирования; платные тарифы начинаются при росте нагрузки/объёма данных.

## AI / генерация сообщений

| Переменная | Что разблокирует | Где получить |
|---|---|---|
| `ANTHROPIC_API_KEY` | Реальная генерация сообщений/классификация через модели Anthropic вместо статичных mock-текстов | console.anthropic.com |
| `OPENAI_API_KEY` | То же самое через модели OpenAI (альтернативный/дополнительный провайдер) | platform.openai.com |

**Стоимость:** оплата по токенам, зависит от объёма генерируемых сообщений в день.
Нужен только один из двух ключей для включения AI-слоя — см. `AI_LAYER.md`.

## Источники сигналов

| Переменная | Источник (`lib/sources.ts`) | Где получить | Заметки |
|---|---|---|---|
| `GOOGLE_PLACES_API_KEY` | `google_places` | Google Cloud Console → включить Places API | Есть бесплатный месячный кредит, далее оплата по использованию |
| `TWOGIS_API_KEY` | `2gis` | Партнёрский портал 2GIS (обращение напрямую) | Публичного прайс-листа на коммерческий API нет — условия уточнять индивидуально |
| `META_ACCESS_TOKEN` | `meta_ads` | Meta for Developers → создать приложение → Marketing API | Чтение Ad Library бесплатно, но строгие rate limits + нужен App Review для прод-доступа |
| `META_AD_ACCOUNT_ID` | `meta_ads` | Meta Business Suite → Ad Account settings | Используется вместе с `META_ACCESS_TOKEN` |
| `INSTAGRAM_ACCESS_TOKEN` | `instagram` | Meta for Developers → Instagram Graph API | Реальный охват ограничен бизнес-аккаунтами, связанными с вашим приложением |
| `GOSZAKUP_API_TOKEN` | `goszakup` | goszakup.gov.kz → портал для разработчиков (регистрация) | Публичный госпортал, формально бесплатный, есть регламентные лимиты обращений |

## Каналы исходящей связи

| Переменная | Что разблокирует | Где получить | Заметки |
|---|---|---|---|
| `WHATSAPP_BUSINESS_TOKEN` | Реальная отправка сообщений лидам через WhatsApp (сейчас `app/api/leads/[id]/messages/route.ts` только симулирует ответ `202`, ничего не отправляя) | Meta for Developers / WhatsApp Business Platform | С июля 2025 действует тарификация **за сообщение** (per-message pricing) — заранее оценить объём отправок в день перед подключением |
| `TELEGRAM_BOT_TOKEN` | Источник `telegram` (мониторинг каналов) + исходящие уведомления через Telegram-бота | @BotFather в Telegram (команда `/newbot`) | Бесплатно. Бот видит только каналы, куда его явно добавили админом |

## Конфигурация приложения (не секреты доступа, но нужны для прода)

| Переменная | Назначение |
|---|---|
| `NEXT_PUBLIC_APP_URL` | Публичный базовый URL задеплоенного приложения — для абсолютных ссылок и вебхук-коллбэков |
| `NODE_ENV` | Стандартный режим окружения Next.js, обычно выставляется автоматически платформой деплоя (Vercel и т.п.) |

## Итоговая рекомендация по порядку подключения

Если/когда будет решение двигаться к реальным данным — разумный порядок (без
давления делать это сейчас):

1. `SUPABASE_URL` + `SUPABASE_ANON_KEY` + `SUPABASE_SERVICE_ROLE_KEY` + `DATABASE_URL`
   — реальная БД вместо mock-массивов.
2. `GOOGLE_PLACES_API_KEY` — первый бесплатный/дешёвый реальный источник сигналов.
3. `TELEGRAM_BOT_TOKEN` — самый низкий порог входа для исходящих уведомлений.
4. `ANTHROPIC_API_KEY` или `OPENAI_API_KEY` — реальная генерация сообщений.
5. `WHATSAPP_BUSINESS_TOKEN` — реальная отправка клиентам (требует больше
   подготовки: лимиты, шаблоны, юридическая проверка — см. `IMPLEMENTATION_ROADMAP.md`
   Фаза 2).
6. Остальные источники (`TWOGIS_API_KEY`, `META_ACCESS_TOKEN`/`META_AD_ACCOUNT_ID`,
   `INSTAGRAM_ACCESS_TOKEN`, `GOSZAKUP_API_TOKEN`) — по мере появления ресурса и
   интереса к конкретному источнику.

Подробный план по фазам — `IMPLEMENTATION_ROADMAP.md`. Подробности по каждому
источнику (стоимость, юридические нюансы) — `SOURCE_CONNECTORS.md`.
